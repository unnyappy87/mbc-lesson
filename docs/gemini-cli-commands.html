Gemini CLI 슬래시 명령어 마스터 가이드

AI 코딩 에이전트를 효율적으로 제어하기 위한 모든 명령어의 상세 사용법입니다.

1. 세션 및 기록 관리

/chat 별칭: /resume
이전 대화를 탐색하고 재개하거나, 현재 대화 상태를 수동으로 저장(체크포인트)하여 나중에 불러올 수 있습니다.
구문
/chat [하위명령어] [태그/파일명]
주요 파라미터
  • list: 사용 가능한 수동 체크포인트 목록을 표시합니다.
  • save <tag>: 현재 세션 상태를 특정 태그로 저장합니다.
  • resume <tag>: 태그된 체크포인트를 로드합니다.
  • share [파일명]: 현재 채팅을 Markdown 또는 JSON으로 내보냅니다.
실행 예시
  • /chat save feature-done
  • /resume list
  • /chat share history.md
/restore
AI 도구에 의해 변경된 파일 내용을 실행 전 상태로 되돌립니다. 잘못된 코드 생성을 즉시 취소할 때 유용합니다.
구문
/restore [tool_call_id]
주요 파라미터
  • [tool_call_id]: (선택 사항) 특정 도구 실행 ID를 지정하여 되돌립니다.
실행 예시
  • /restore (최근 체크포인트 목록 확인)
/rewind
대화 기록을 시각적으로 탐색하며 과거의 특정 시점으로 채팅 및 파일 상태를 되돌립니다.
단축키
Esc 키를 두 번 연속으로 누름

2. 워크스페이스 및 컨텍스트

/memory
GEMINI.md 파일에서 로드된 계층적 지침(Instructional Context)을 관리합니다. AI가 프로젝트의 규칙을 더 잘 이해하도록 돕습니다.
구문
/memory <하위명령어>
하위 명령어
  • list / show: 현재 적용된 메모리 내용을 확인합니다.
  • refresh: GEMINI.md 파일을 다시 읽어옵니다.
  • add <텍스트>: 일시적인 지침을 추가합니다.
실행 예시
  • /memory refresh
  • /memory show
/directory 별칭: /dir
현재 워크스페이스 외에 AI가 접근할 수 있는 추가 디렉토리를 지정합니다.
구문
/directory <add | show> [경로]
실행 예시
  • /directory add ../shared-utils
  • /dir show
/compress
현재까지의 대화 내용을 요약하여 컨텍스트 윈도우(토큰)를 확보합니다. 긴 대화 시 성능 유지에 필수적입니다.
구문
/compress

3. 도구 및 확장 기능

/skills
"에이전트 기술(Agent Skills)"을 관리합니다. 특정 분야의 전문가 워크플로우를 활성화하거나 비활성화할 수 있습니다.
구문
/skills <list | enable | disable | reload> [기술이름]
실행 예시
  • /skills list all
  • /skills enable code-reviewer
/mcp
Model Context Protocol 서버를 관리합니다. 외부 도구 및 데이터 소스와의 연결을 제어합니다.
구문
/mcp <list | reload | desc | auth> [서버이름]
실행 예시
  • /mcp reload (모든 MCP 서버 재시작)
  • /mcp list

4. 시스템 및 설정

/settings
CLI의 모든 환경 설정을 변경할 수 있는 대화형 에디터를 실행합니다.
/model
사용 중인 AI 모델을 확인하거나 즉시 변경합니다.
구문
/model <manage | set> [모델명] [--persist]
실행 예시
  • /model set gemini-2.0-flash --persist
기타 유용한 명령어
  • /copy: 마지막 AI 답변을 클립보드에 즉시 복사합니다.
  • /quit --delete: 세션 기록을 완전히 삭제하고 종료합니다.
  • /vim: 입력창에서 Vim 방식의 편집 모드를 켭니다/끕니다.
  • /bug <설명>: 버그를 발견하면 즉시 GitHub 이슈로 제출합니다.
