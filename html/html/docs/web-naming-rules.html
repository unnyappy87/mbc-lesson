🌈 웹개발 파일 & 폴더 네이밍 규칙

초등학생도 쉽게 이해하는 네이밍 규칙 가이드!

1️⃣ 소문자만 사용해요!

파일명과 폴더명은 모두 소문자로 써요. 대문자와 소문자를 섞으면 컴퓨터가 헷갈려할 수 있어요.

✅ about.html    ✅ my-style.css    ✅ images/
❌ About.HTML    ❌ MyStyle.CSS    ❌ IMAGES/
2️⃣ 띄어쓰기는 하이픈(-)으로!

띄어쓰기 대신 하이픈(-)을 사용해요. 띄어쓰기는 웹 주소에서 이상하게 보일 수 있어요.

✅ my-home-page.html    ✅ contact-us.css    ✅ my-profile.jpg
❌ my home page.html    ❌ contact us.css    ❌ my profile.jpg
3️⃣ 언더스코어(_)는 폴더용으로!

폴더명에는 언더스코어(_)를 써도 괜찮아요. 하지만 파일명에는 하이픈(-)을 쓰는 게 더 예뻐요.

✅ assets_images/    ✅ old_files/    ✅ backup_data/
❌ assets-images/ (폴더는 _가 더 좋아요)    ❌ old-files/
4️⃣ 특수문자는 피해요!

한글, !, @, #, $, %, &, *, (, ), +, =, ?, /, \ 같은 특수문자는 쓰지 마세요. 컴퓨터가 못 알아들을 수 있어요!

❌ 내홈페이지.html    ❌ file@home!.css    ❌ image#1.jpg    ❌ what?.js
✅ my-homepage.html    ✅ file-home.css    ✅ image-1.jpg
5️⃣ 의미있는 이름으로!

파일이 뭔지 알 수 있게 의미있는 이름을 지어요. a.html, b.css 같은 이름은 나중에 까먹어요.

✅ index.html (첫 페이지)    ✅ main-style.css (메인 스타일)    ✅ logo.png (로고 이미지)
❌ a.html    ❌ 1.css    ❌ x.jpg    ❌ new.html (무슨 new?)
6️⃣ 확장자는 꼭 붙여요!

파일의 종류를 알려주는 확장자를 빠뜨리지 마세요. 컴퓨터가 어떤 파일인지 알아야 해요.

  • HTML 파일: .html (예: index.html)
  • CSS 파일: .css (예: style.css)
  • JavaScript 파일: .js (예: script.js)
  • 이미지 파일: .jpg, .png, .gif, .svg
7️⃣ 폴더는 깔끔하게 정리해요!

비슷한 파일끼리 폴더에 모아두면 찾기 쉬워요. 아래는 예쁜 폴더 구조예요:

📁 my-website/          <-- 웹사이트 메인 폴더
├── index.html              <-- 첫 페이지 (꼭 있어야 해요!)
├── about.html              <-- 소개 페이지
├── contact.html            <-- 연락처 페이지
├── 📁 css/                 <-- 스타일 파일 모음
│   ├── main-style.css
│   └── reset.css
├── 📁 js/                  <-- 자바스크립트 파일 모음
│   ├── main-script.js
│   └── animation.js
├── 📁 images/              <-- 이미지 파일 모음
│   ├── logo.png
│   ├── hero-banner.jpg
│   └── icon-menu.svg
└── 📁 fonts/               <-- 글꼴 파일 모음
    └── my-font.woff2
💡 보너스 팁!
  • 숫자로 시작하지 마세요 (❌ 1page.html → ✅ page-1.html)
  • 이름은 짧고 간단하게! (❌ this-is-my-very-long-file-name-for-home-page.html)
  • 버전 표시는 날짜로! (✅ report-2024-05-19.pdf)
  • index.html은 항상 메인 폴더에!
  • 파일명에 공백 2개 이상 넣지 마세요
